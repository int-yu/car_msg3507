# 遥测调参升级设计（分支 update-遥测）

## 背景与问题

现有调试链路（UART1 命令 + 遥测 CSV + car_debug.html）能看状态，但调 PID 的闭环体验差：

1. **所有控制参数是编译期 `#define`**（`MotionWheel.h`、`MotionStraight.h`、`Nav.h`、`MotionLine.h` 顶部），改一次参数就要重编译烧录，一轮调参要几分钟。
2. **遥测里没有"目标值"**：只有实测 LV/RV/yaw，看不到目标轮速、输出 PWM、转向目标角/误差、巡线误差，无法画"目标 vs 实测"的阶跃响应曲线。
3. **没有干净的激励命令**：`F` 带梯形速度规划，不是阶跃；巡线只能改 main.c 换 Mission 状态图才能启动。
4. **陀螺仪尺度标定（`Heading_ScaleCalib*`）和里程标定（`Odometry_CountsPerMM`）固件接口早已存在，但没有串口命令**，用不上。
5. 网页曲线每条独立归一化，目标和实测不同刻度，没法直接对比；一次试验的数据混在连续日志里，不能单独导出给人或 AI 分析。

用户原始想法（保留）：调各 PID 有专属命令、只传对应数据以提高频率、开始前清日志获得专属数据、可视化好、方便 AI 帮忙调参。

对用户想法的两处修正：

- **"弧线巡线 PID"不新建独立控制器**。当前巡线是离散权重差速（刻意不用 PID，README 3.5），推倒重来风险大。改为：巡线调整比例 `lra` 变为运行时可调，并新增可选的权重微分阻尼 `lkd`（默认 0，行为与现状完全一致）；弧线抖动可用 lkd 抑制。这覆盖"直线/弧线巡线"的调参需求而不动控制结构。
- **"每种 PID 一条专属遥测命令"不在固件做**。固件只提供字段掩码原语（M 命令扩展），"预设掩码+频率+清屏+激励+录制"的编排全部放网页——固件保持契约简单，网页可以随时改预设而不用重烧。

## 方案总览

### A. 固件：运行时参数系统（K 命令）

新模块 `Application/Debug/Param.c/.h`：参数注册表 `{名称, get, set, min, max}`，串口命令：

- `K?` — 列出全部参数：每行 `K<id>=<值> <名> [<min>,<max>]`
- `K<id>?` — 读单个：`OK K<id>=<值>`
- `K<id>=<float>` — 写入（支持小数/负号），越界回 `ERR K RANGE MIN=.. MAX=..`

参数表（id 固定，掉电恢复 `#define` 默认值）：

| id | 名 | 对应常量 | 模块 |
|---|---|---|---|
| 1 | wkp | MOTION_WHEEL_KP | 轮速 PI |
| 2 | wki | MOTION_WHEEL_KI | 轮速 PI |
| 3 | wil | MOTION_WHEEL_INTEGRAL_LIMIT | 轮速 PI |
| 4 | wff | MOTION_WHEEL_FEEDFORWARD_PWM_PER_MMPS | 轮速前馈 |
| 5 | wsf | MOTION_WHEEL_STATIC_FRICTION_PWM | 轮速前馈 |
| 6 | skp | MOTION_STRAIGHT_HEADING_KP | 直线航向 PD |
| 7 | skd | MOTION_STRAIGHT_HEADING_KD | 直线航向 PD |
| 8 | sac | MOTION_STRAIGHT_ACCELERATION_MMPS2 | 直线规划 |
| 9 | lra | MOTION_LINE_MAX_ADJUST_RATIO | 巡线 |
| 10 | lkd | （新增）巡线权重微分阻尼，默认 0 | 巡线 |
| 11 | nvx | NAV_MAX_TURN_SPEED_MMPS | 转向 |
| 12 | nvn | NAV_MIN_TURN_SPEED_MMPS | 转向 |
| 13 | nsa | NAV_SLOWDOWN_ANGLE_DEG | 转向 |
| 14 | ntl | NAV_ANGLE_TOLERANCE_DEG | 转向 |
| 15 | gsc | 陀螺仪尺度因子（Heading_Get/SetScale） | 航向 |
| 16 | cpm | Odometry_CountsPerMM | 里程 |

实现方式：各模块把上述 `#define` 的**使用点**改为同名运行时变量（初值 = 原 `#define`），PID 增益类参数写入后通过 apply 钩子调用 `PID_SetTunings` 立即生效。头文件 `#define` 保留作为默认值和上电基准；调好后人工把数值写回 `#define` 固化。

解析器扩展：`BluetoothDebug` 的字符状态机对 `K` 命令启用文本参数缓冲（接受 `0-9 . - = ?`），其余命令解析路径不变。

### B. 固件：遥测新字段（掩码扩为 16 位）

| 掩码位 | 列名 | 内容 |
|---|---|---|
| 0x40 | `TL,TR` | MotionWheel 本拍目标轮速 mm/s |
| 0x80 | `PL,PR` | MotionWheel 最终输出 PWM |
| 0x100 | `navT,navE` | Nav 目标航向角与当前角误差（度） |
| 0x200 | `lerr` | 巡线离散权重误差 −6~+6 |

`TELEMETRY_FIELD_ALL = 0x3FF`，`M` 范围扩到 1~1023。行长估算改用 uint16 累加；全字段安全上限降到约 14 Hz（调参预设用子集，50 Hz 无压力）。

### C. 固件：新命令

| 命令 | 作用 |
|---|---|
| `W<±mmps>` | 闭环恒速模式（MotionManager 新增 MODE_SPEED）：双轮同目标速度、无规划斜坡、无航向修正——轮速 PI 的**标准阶跃激励**。同模式下再发 W 直接改目标（不复位 PID，可做链式阶跃）；`W0` 停止释放电机 |
| `N<mmps>` | 直接启动巡线（MotionManager_StartLine），不再需要改 main.c 换 Mission；`N0` 停止 |
| `E1` / `E0` | 陀螺仪尺度标定 开始 / 取消（包装 Heading_ScaleCalibStart/Cancel） |
| `Y<n>` | 原地转 n 整圈后结束标定，回 `OK Y SCALE=<新因子> RAW=<积分角>` |

安全语义与现有命令一致：W/N 在他模式忙时回 `ERR BUSY`；`C0` 全局停车覆盖所有新模式；W0/N0 只停自己的模式。

### D. 网页 car_debug.html

1. **参数面板**：连接后自动 `K?` 拉全表；表格内改值回车/按钮写入；显示上电默认值；一键"存到浏览器 / 回写小车"（localStorage，解决掉电丢参数的痛点）。
2. **调参试验**（核心工作流）：四种试验各带预设——
   - 轮速：掩码 mode+speed+target+pwm，50 Hz，激励 `W<幅值>`，定时后自动 `W0`
   - 直线：掩码 +yaw+distance，激励 `F<距离>`
   - 转向：掩码 yaw+nav+target+speed+mode，激励 `T<角度>`
   - 巡线：掩码 sensor+lerr+speed+target+mode，激励 `N<速度>`，定时/丢线结束
   流程：`G0` 静默 → 发预设 M/G → **清终端+清曲线（专属数据）** → 记录参数快照 → 发激励 → 检测 mode 回 IDLE（或超时/手动结束）→ 自动收尾成一条**试验记录**。
3. **试验记录列表**：每条含环路、时间、激励、参数快照、样本数；可导出 CSV、可一键复制 **AI 调参包**（Markdown：控制结构说明 + 当前参数 + 激励描述 + 列说明 + CSV 数据，直接粘给 AI 要参数建议）。
4. **标定向导**：陀螺（E1 → 提示转 N 圈 → Y n → 显示新 scale，并可写 gsc 固化到浏览器存档）；里程（记起点 → 跑 F1000 → 记终点 → 填卷尺实测 → 算出新 cpm → 一键写入 K16）。
5. **曲线升级**：按量纲分组共享刻度（mm/s 组：LV/RV/TL/TR；PWM 组：PL/PR；角度组：yaw/navT/navE；mm 组：LD/RD），目标 vs 实测终于同刻度可比；新增暂停/继续按钮；新字段进掩码选择器。

## 明确不做（本分支）

- 巡线改成连续 PID 控制器（结构重构，另开分支）
- 参数写 MCU Flash 持久化（用网页 localStorage 回写代替）
- 串口控制无刷云台（固件中尚无该模块，用户已确认跳过）
- 舵机相关（用户要求忽略，保留现状）
