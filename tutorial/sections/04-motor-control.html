模块 04：电机驱动与轮速闭环

上层说“左轮 200mm/s、右轮 180mm/s”，电机听不懂毫米每秒，只认识方向引脚和 PWM。本模块就是翻译链：先用编码器实测速度做 PID 修正，再加上前馈和差速修正，最后把带正负号的 PWM 命令变成 TB6612 的方向电平与 20kHz 占空比。

1. 先看它在整车中的位置

一句话职责
把左右轮目标速度转换为受限的左右 PWM，并用编码器速度反馈减小“目标速度－实际速度”的误差。
上游输入
MotionStraight、MotionLine 或 Nav 产生的左右目标速度、附加差速修正 trim，以及 App 传下来的真实 dt。
反馈输入
Odometry 根据左右编码器算出的 Odometry_GetSpeedL/R()，单位 mm/s。
下游输出
左右带符号 PWM 命令（逻辑范围 -1000～1000）、电机方向 GPIO，以及 TIMG8 两路 PWM 比较值。
依赖模块
Odometry、TI DriverLib/SysConfig、GPIO、TIMG8 PWM 定时器和 TB6612 电机驱动器。
被谁调用
MotionManager 统一选择 MotionStraight、MotionLine、Nav 等上层运动模块；这些模块共用 MotionWheel，MotionWheel 再调用 Motor 和 PWM。
总体位置
Mission → MotionManager → MotionStraight / MotionLine / Nav → MotionWheel → PID + 前馈 → Motor → PWM/GPIO → 电机。

2. 初学者需要先知道

  • 脉宽调制：让引脚快速开关，用“每个周期有多少时间为有效电平”控制平均功率。像快速反复开关水龙头，开得越久，平均水流越大。
  • 占空比：有效时间占整个 PWM 周期的比例。本项目用 0～1000 表示 0%～100%，500 就是约 50%。
  • H 桥：通过两根方向输入改变电机两端电压方向，因此同一个电机可以正转、反转、滑行或主动刹车。TB6612 的 A、B 是两套电机通道。
  • 闭环控制：不只“发一个命令”，还读取实际结果并纠偏。这里目标轮速是 setpoint，编码器轮速是 measure，差值是 error。
  • 比例－积分－微分控制：P 看当前误差，I 累加过去误差，D 看误差变化速度。三项加权后成为反馈修正量。
  • 前馈：根据目标速度先猜一个基础 PWM，不必等误差出现才出力；反馈 PID 再修正电池、电机、负载造成的偏差。
  • 静摩擦补偿：目标不为零时额外加一小段同方向 PWM，帮助电机跨过“命令很小但轮子不动”的死区。当前左右轮分别使用 19.884 和 22.205 PWM，承认两套机械系统的起步阻力不同。
  • 饱和/限幅：把过大的值截到允许边界，防止命令超过 PWM 硬件范围。
  • 结构体指针：PID_t *pid 指向一套 PID 参数与历史状态；pid->integral 就是修改那套对象里的积分成员。
  • 只读指向对象：const MotionWheel_Command_t *command 表示函数可读取命令结构体，但不应通过该指针改写它。
  • 时间增量：本次控制距离上次控制经过的秒数。积分要乘 dt，微分要除 dt，否则控制效果会随循环快慢改变。
  • 有限数：不是 NaN，也不是正负无穷。控制参数或传感数据一旦出现这些异常浮点值，比较和输出会变得不可预测，因此代码先用 isfinite() 拒绝它们。

3. 文件导读

文件角色单位/边界
Hardware/Motor/PWM.h/.c最底层双通道 PWM 定时器驱动，把 0～1000 占空比刻度转换为 TIMG8 比较值。无方向概念，只接收非负强度。
Hardware/Motor/Motor.h/.c处理正负号、左右物理接线、H 桥方向 GPIO、停止与主动刹车。逻辑命令 -1000～1000。
Application/Control/PID.h/.c通用位置式 PID 算法，保存 Kp/Ki/Kd、积分和上次误差。本身不知道 mm/s 或 PWM，单位由调用者决定。
Application/Control/MotionWheel.h/.c双轮速度控制总入口：校验参数，读取左右实测速度，组合前馈、PID 和 trim，再驱动 Motor。输入速度 mm/s，输出 PWM。

4. 逐行讲解

4.1 Hardware/Motor/PWM.h：底层 PWM 接口

#ifndef __PWM_H
#define __PWM_H

/* 电机双通道底层 PWM 定时器驱动。 */

#include <stdint.h>

#define PWM_MAX_DUTY    1000U

void PWM_Init(void);
void PWM_SetCompareA(uint16_t Compare);
void PWM_SetCompareB(uint16_t Compare);

#endif
  1. 第 1～2、14 行：头文件保护，避免同一声明被重复展开。
  2. 第 4 行：说明该层只管定时器的两个 PWM 通道，不判断小车前进还是后退。
  3. 第 6 行：引入 uint16_t；它是 16 位无符号整数。
  4. 第 8 行：项目把满占空比分成 1000 份。U 表示无符号整数常量。
  5. 第 10 行：初始化两路占空比并启动 PWM 计数器。
  6. 第 11～12 行：分别设置驱动器 A、B 通道。参数名虽叫 Compare，调用者实际传的是 0～1000 的占空比刻度，函数内部才换算成寄存器比较值。

4.2 Hardware/Motor/PWM.c：占空比换算

#include "Hardware/Motor/PWM.h"
#include "ti_msp_dl_config.h"

#define MOTOR_PWM_PERIOD_TICKS 1600U

static uint16_t PWM_DutyToCompare(uint16_t duty)
{
    uint32_t highTicks;
    if (duty > PWM_MAX_DUTY) duty = PWM_MAX_DUTY;
    highTicks = ((uint32_t)duty * MOTOR_PWM_PERIOD_TICKS) / PWM_MAX_DUTY;
    return (uint16_t)(MOTOR_PWM_PERIOD_TICKS - highTicks);
}
  1. 第 1～2 行：取得本模块声明和 SysConfig 生成的定时器实例、通道编号及 DriverLib 函数。
  2. 第 4 行：一个 PWM 周期包含 1600 个计数。当前 32MHz 定时器时钟下，32,000,000 / 1600 = 20,000Hz，即 20kHz。
  3. 第 6～7 行：文件私有转换函数，只允许本 PWM.c 使用。
  4. 第 8 行：用 32 位临时变量保存乘法结果；最大 1000 × 1600 = 1,600,000，超过 16 位上限 65535。
  5. 第 9 行：调用者即使传入大于 1000 的值，也先截成 1000，避免超出 100% 占空比。
  6. 第 10 行：按比例换算高电平计数：duty × 1600 / 1000。整数除法会舍去小数部分。
  7. 第 11～12 行：当前定时器输出方式要求写入“周期计数－有效计数”，所以 duty 越大，寄存器 compare 反而越小；最后转回 16 位。

500800800约 50% 输出
100016000100% 输出
 
void PWM_Init(void)
{
    PWM_SetCompareA(0U);
    PWM_SetCompareB(0U);
    DL_TimerG_startCounter(MOTOR_PWM_INST);
}

void PWM_SetCompareA(uint16_t compare)
{
    DL_TimerG_setCaptureCompareValue(
        MOTOR_PWM_INST, PWM_DutyToCompare(compare), GPIO_MOTOR_PWM_C0_IDX);
}

void PWM_SetCompareB(uint16_t compare)
{
    DL_TimerG_setCaptureCompareValue(
        MOTOR_PWM_INST, PWM_DutyToCompare(compare), GPIO_MOTOR_PWM_C1_IDX);
}
  1. 第 14～18 行：先把 A、B 两路设为 0% 再启动计数器，防止初始化瞬间输出意外脉冲。
  2. 第 21～25 行：A 通道先做 duty→compare 换算，再把结果写入 TIMG8 的 C0 比较通道；本车物理右电机接 A。
  3. 第 27～31 行：B 通道做同样处理，但写入 C1；本车物理左电机接 B。

4.4 Hardware/Motor/Motor.c：接线映射、正负号与限幅

#include "Hardware/Motor/Motor.h"
#include "Hardware/Motor/PWM.h"
#include "ti_msp_dl_config.h"

/* 实测: 物理左电机接在驱动器 B 通道、物理右电机接在 A 通道(接线左右反了)。
 * 故左轮走 B 通道(前进符号 +1)、右轮走 A 通道(前进符号 -1)。
 * 交叉的后果: 左环算的 PWM 驱动到右电机 → 巡线/航向差速全反 → 发 s 冲出去/打转。
 * 此处把左右各自绑定到正确的物理通道(与 ti_24_h 一致)。 */
#define LEFT_MOTOR_DIR_SIGN  (-1)   /* 左电机 = B 通道 */
#define RIGHT_MOTOR_DIR_SIGN (+1)   /* 右电机 = A 通道 */

static int16_t Motor_ClampPWM(int16_t pwm)
{
    if (pwm > (int16_t)PWM_MAX_DUTY) return (int16_t)PWM_MAX_DUTY;
    if (pwm < -(int16_t)PWM_MAX_DUTY) return -(int16_t)PWM_MAX_DUTY;
    return pwm;
}
  1. 第 1～3 行：取得带方向接口、底层 PWM 接口和 GPIO/定时器配置。
  2. 第 5～8 行：记录实车接线事实：软件“左轮”必须发到 B，软件“右轮”必须发到 A。若左右绑定交叉，外层算出的差速会作用到错误车轮，方向控制会整体反常。
  3. 第 9～10 行：左轮逻辑正号与硬件方向一致；右轮因电机安装/接线方向相反，要乘 -1。这样上层仍可统一认为“两轮正数都表示小车前进”。
  4. 第 12～13 行：文件私有限幅函数，输入输出都是带符号 16 位 PWM。
  5. 第 14～15 行：大于 1000 截成 1000，小于 -1000 截成 -1000。显式强制类型转换使宏值与 int16_t 比较一致。
  6. 第 16～17 行：范围内的值原样返回。
static void Motor_WritePin(GPIO_Regs *port, uint32_t pin, uint8_t high)
{
    if (high != 0U) DL_GPIO_setPins(port, pin);
    else DL_GPIO_clearPins(port, pin);
}

void Motor_Init(void)
{
    PWM_Init();
    Motor_StopAll();
}

void Motor_SetLeftPWM(int16_t pwm)   /* 物理左电机 = 驱动器 B 通道(BIN/CompareB) */
{
    int16_t value = Motor_ClampPWM((int16_t)(pwm * LEFT_MOTOR_DIR_SIGN));
    Motor_WritePin(BOARD_OUTPUTS_MOTOR_BIN1_PORT, BOARD_OUTPUTS_MOTOR_BIN1_PIN, value >= 0);
    Motor_WritePin(BOARD_OUTPUTS_MOTOR_BIN2_PORT, BOARD_OUTPUTS_MOTOR_BIN2_PIN, value < 0);
    PWM_SetCompareB((uint16_t)((value < 0) ? -value : value));
}
  1. 第 19～20 行：辅助函数接收 GPIO 端口指针、引脚掩码和逻辑高低。GPIO_Regs *port 指向一组硬件寄存器。
  2. 第 21～23 行：high 非零就置位引脚，否则清零；集中封装后，左右轮方向代码更容易读。
  3. 第 25～29 行：先安全启动两路 PWM，再明确执行一次停止，使占空比与方向引脚都进入已知状态。
  4. 第 31～33 行：左轮使用 B 通道；先乘接线方向符号 +1，再限制在 ±1000。
  5. 第 34～35 行：value 非负时 BIN1 高、BIN2 低；value 为负时相反。比较表达式在 C 中产生 0 或 1，可直接作为 high 参数。
  6. 第 36～37 行：PWM 只需要强度，所以三目运算取绝对值，再转为无符号数送给 B 通道。方向已经由 BIN1/BIN2 表达。
void Motor_SetRightPWM(int16_t pwm)  /* 物理右电机 = 驱动器 A 通道(AIN/CompareA) */
{
    int16_t value = Motor_ClampPWM((int16_t)(pwm * RIGHT_MOTOR_DIR_SIGN));
    Motor_WritePin(BOARD_OUTPUTS_MOTOR_AIN1_PORT, BOARD_OUTPUTS_MOTOR_AIN1_PIN, value >= 0);
    Motor_WritePin(BOARD_OUTPUTS_MOTOR_AIN2_PORT, BOARD_OUTPUTS_MOTOR_AIN2_PIN, value < 0);
    PWM_SetCompareA((uint16_t)((value < 0) ? -value : value));
}

void Motor_SetPWM(int16_t leftPWM, int16_t rightPWM)
{
    Motor_SetLeftPWM(leftPWM);
    Motor_SetRightPWM(rightPWM);
}

void Motor_StopAll(void)
{
    PWM_SetCompareA(0U);
    PWM_SetCompareB(0U);
    DL_GPIO_clearPins(GPIOA, BOARD_OUTPUTS_MOTOR_AIN1_PIN | BOARD_OUTPUTS_MOTOR_AIN2_PIN);
    DL_GPIO_clearPins(GPIOB, BOARD_OUTPUTS_MOTOR_BIN1_PIN | BOARD_OUTPUTS_MOTOR_BIN2_PIN);
}
  1. 第 39～41 行：右轮绑定 A 通道，并乘 -1 修正实车方向；上层给正数，底层 value 会变负以产生物理前进。
  2. 第 42～44 行：按 value 正负设置 AIN1/AIN2，再把绝对值送入 A 通道 PWM。
  3. 第 47～51 行：双轮接口只是依次调用左右轮接口，因此左右映射、限幅和方向修正都不会重复实现。
  4. 第 53～56 行：普通停止先把 A、B 两路 PWM 都设为 0。
  5. 第 57～58 行：| 是按位或，把同一端口的两个引脚掩码合并，一次清低 AIN1/AIN2 或 BIN1/BIN2。
  6. 第 59 行：普通停止完成。它不等于主动刹车。
void Motor_Brake(void)
{
    DL_GPIO_setPins(GPIOA, BOARD_OUTPUTS_MOTOR_AIN1_PIN | BOARD_OUTPUTS_MOTOR_AIN2_PIN);
    DL_GPIO_setPins(GPIOB, BOARD_OUTPUTS_MOTOR_BIN1_PIN | BOARD_OUTPUTS_MOTOR_BIN2_PIN);
    PWM_SetCompareA(PWM_MAX_DUTY);
    PWM_SetCompareB(PWM_MAX_DUTY);
}

void Motor_Run(int16_t leftSpeed, int16_t rightSpeed) { Motor_SetPWM(leftSpeed, rightSpeed); }
void Motor_Forward(int16_t speed) { Motor_SetPWM(speed, speed); }
void Motor_Backward(int16_t speed) { Motor_SetPWM(-speed, -speed); }
void Motor_TurnLeft(int16_t speed) { Motor_SetPWM(0, speed); }
void Motor_TurnRight(int16_t speed) { Motor_SetPWM(speed, 0); }
void Motor_SpinLeft(int16_t speed) { Motor_SetPWM(-speed, speed); }
void Motor_SpinRight(int16_t speed) { Motor_SetPWM(speed, -speed); }
void Motor_Stop(void) { Motor_StopAll(); }
  1. 第 61～64 行：主动刹车把每个 H 桥通道的两根方向线同时置高，这是 TB6612 的短刹状态。
  2. 第 65～67 行：两路 PWM 设为满值，让刹车状态持续作用；具体只应短时间使用，由更高层 MotionManager 管理。
  3. 第 69 行：Motor_Run 是双轮设置的别名。
  4. 第 70～71 行：前进给两轮同号正值，后退给两轮同号负值。底层的左右方向修正保证物理方向一致。
  5. 第 72～73 行：普通左/右转让内侧轮为 0、外侧轮前进，车体绕靠近停止轮的位置转弯。
  6. 第 74～75 行：原地旋转让两轮大小相同、方向相反，车体中心附近转动。
  7. 第 76 行：Motor_Stop 只是调用普通停止接口。

4.5 Application/Control/PID.h：一套 PID 的参数与记忆

#ifndef APPLICATION_CONTROL_PID_H
#define APPLICATION_CONTROL_PID_H

typedef struct
{
    float Kp;
    float Ki;
    float Kd;
    float integral;
    float prevError;
    float outMax;
    float integralMax;
} PID_t;

void PID_Init(PID_t *pid, float kp, float ki, float kd,
              float outMax, float integralMax);
void PID_SetTunings(PID_t *pid, float kp, float ki, float kd);
void PID_Reset(PID_t *pid);
float PID_Update(PID_t *pid, float setpoint, float measure, float dt);

#endif
  1. 第 1～2、21 行：头文件保护。
  2. 第 4～5、13 行：定义 PID_t 结构体；每个车轮各有一份，所以左右积分和历史误差不会混在一起。
  3. 第 6 行：Kp 乘当前误差；越大，立刻纠偏越强，也越容易振荡。
  4. 第 7 行：Ki 乘误差随时间的累积，用于消除长期小偏差，但过大会积累过头。
  5. 第 8 行：Kd 乘误差变化速度，用于抑制快速变化，但对噪声敏感。
  6. 第 9～10 行：integral 和 prevError 是控制器的“记忆”，每次 Update 都会改变。
  7. 第 11～12 行：分别限制最终输出绝对值与积分状态绝对值。
  8. 第 15～16 行：初始化一份 PID：通过指针填写参数并清空历史。
  9. 第 17 行：运行中只更换 Kp/Ki/Kd，不自动清空历史状态。
  10. 第 18 行：只清空积分和上次误差，不改变调参值。
  11. 第 19 行：输入目标、测量和时间间隔，返回一次 PID 输出。

4.6 Application/Control/PID.c：P、I、D 的真实计算

#include "PID.h"

/* 通用位置式 PID —— 见 PID.h 接口说明 */

static float PID_Clamp(float v, float limit)
{
    if (limit > 0.0f)
    {
        if (v > limit)  { return limit; }
        if (v < -limit) { return -limit; }
    }
    return v;
}
  1. 第 1 行：引入同目录 PID 接口；工程的头文件搜索路径使 "PID.h" 能被找到。
  2. 第 3 行：这是通用“位置式”PID：每次直接算完整输出，而不是只算输出增量。
  3. 第 5～6 行：文件私有浮点限幅函数。
  4. 第 7～11 行：只有 limit 大于 0 才启用对称限幅；超过 +limit 返回 +limit，低于 -limit 返回 -limit。
  5. 第 12～13 行：未越界或 limit≤0 时原值返回。也就是说 0 在这个通用函数中表示“不启用限幅”。
void PID_Init(PID_t *pid, float kp, float ki, float kd, float outMax, float integralMax)
{
    pid->Kp = kp;
    pid->Ki = ki;
    pid->Kd = kd;
    pid->outMax = outMax;
    pid->integralMax = integralMax;
    pid->integral = 0.0f;
    pid->prevError = 0.0f;
}

void PID_SetTunings(PID_t *pid, float kp, float ki, float kd)
{

float PID_Update(PID_t *pid, float setpoint, float measure, float dt)
{
    float error = setpoint - measure;
    float deriv = 0.0f;
    float out;

    /* 积分(带抗饱和限幅) */
    pid->integral += error * dt;
    pid->integral = PID_Clamp(pid->integral, pid->integralMax);

    /* 微分 */
    if (dt > 0.0f)
    {
        deriv = (error - pid->prevError) / dt;
    }
    pid->prevError = error;

    out = pid->Kp * error + pid->Ki * pid->integral + pid->Kd * deriv;
    return PID_Clamp(out, pid->outMax);
}
  1. 第 39～43 行：误差定义为目标减实测；微分先设 0，out 稍后保存总和。若目标 200、实测 180，error=+20，控制器需要加力。
  2. 第 45～47 行：积分按 integral += error × dt 累积，再限制历史总量。这是简单的积分抗饱和，防止长时间大误差把积分堆得过大。
  3. 第 49～53 行：dt>0 才计算误差变化率，避免除以 0；公式是“本次误差－上次误差”除以经过秒数。
  4. 第 54 行：保存本次误差，供下一次算 D。
  5. 第 56 行：位置式 PID 总输出为 Kp×error + Ki×integral + Kd×deriv。
  6. 第 57～58 行：最终输出再按 outMax 对称限幅后返回。

当前轮速反馈部分实际上是 P 控制。MotionWheel 把左右 Ki 配成 0，也把 Kd 固定传成 0，所以 PID 的反馈修正只有 P 项；最终电机命令并不只是 P，还包含左右各自的前馈和静摩擦补偿。启用 Ki 时必须同时给正的积分上限；Kd 若要启用，还需给 MotionWheel 增加清晰的配置入口并重新实车调参。

static float MotionWheel_GetFeedforward(
    float targetSpeedMMps, float feedforwardPWMPerMMps,
    float staticFrictionPWM)
{
    float output;

    if (fabsf(targetSpeedMMps) < 0.001f)
    {
        return 0.0f;
    }

    output = targetSpeedMMps * feedforwardPWMPerMMps;
    output += (targetSpeedMMps > 0.0f) ?
                  staticFrictionPWM : -staticFrictionPWM;
    return output;
}

static int16_t MotionWheel_ToMotorCommand(float value)
{

    return (int16_t)value;
}

4.10 MotionWheel_Update()：mm/s 到 PWM 的完整闭环

MotionWheel_Result_t MotionWheel_Update(
    const MotionWheel_Command_t *command, float dt)
{
    float leftFeedbackPWM;
    float rightFeedbackPWM;

    if (s_configured == 0U)
    {
        return MOTION_WHEEL_RESULT_NOT_CONFIGURED;
    }
    if ((command == NULL) || (!isfinite(dt)) || (dt <= 0.0f) ||
        (!isfinite(command->targetSpeedLMMps)) ||
        (!isfinite(command->targetSpeedRMMps)) ||
        (!isfinite(command->trimLPWM)) || (!isfinite(command->trimRPWM)))
    {
        MotionWheel_Stop();
        return MOTION_WHEEL_RESULT_INVALID_ARGUMENT;
    }
    if (Odometry_CountsPerMM <= 0.001f)
    {
        MotionWheel_Stop();
        return MOTION_WHEEL_RESULT_ODOMETRY_INVALID;
    }

    s_targetSpeedLMMps = command->targetSpeedLMMps;
    s_targetSpeedRMMps = command->targetSpeedRMMps;

    leftFeedbackPWM = PID_Update(
        &s_leftPID, command->targetSpeedLMMps, Odometry_GetSpeedL(), dt);
    rightFeedbackPWM = PID_Update(
        &s_rightPID, command->targetSpeedRMMps, Odometry_GetSpeedR(), dt);

    s_leftCommandPWM = MotionWheel_GetFeedforward(
        command->targetSpeedLMMps,
        MotionWheel_TuneLeftFeedforwardPWMPerMMps,
        MotionWheel_TuneLeftStaticFrictionPWM) +
        leftFeedbackPWM + command->trimLPWM;
    s_rightCommandPWM = MotionWheel_GetFeedforward(
        command->targetSpeedRMMps,
        MotionWheel_TuneRightFeedforwardPWMPerMMps,
        MotionWheel_TuneRightStaticFrictionPWM) +
        rightFeedbackPWM + command->trimRPWM;
    s_leftCommandPWM = MotionWheel_Clamp(
        s_leftCommandPWM, MOTION_WHEEL_MAX_COMMAND_PWM);
    s_rightCommandPWM = MotionWheel_Clamp(
        s_rightCommandPWM, MOTION_WHEEL_MAX_COMMAND_PWM);

    Motor_SetPWM(MotionWheel_ToMotorCommand(s_leftCommandPWM),
                 MotionWheel_ToMotorCommand(s_rightCommandPWM));
    return MOTION_WHEEL_RESULT_OK;
}
  1. 第 140～157 行：检查配置、空指针、dt 与四个命令字段；动态输入异常时主动停车并返回错误。
  2. 第 158～162 行：编码器计数/mm 无效时，实测 mm/s 没有意义，因此同样停车。
  3. 第 164～170 行：先保存左右目标供遥测使用，再让左 PID 对左实测、右 PID 对右实测，两个历史状态完全独立。
  4. 第 172～181 行：左轮使用 lwff/lwsf，右轮使用 rwff/rwsf。每侧公式都是 speed×ff + sign(speed)×sf + PI + trim。
  5. 第 182～189 行：两侧分别夹到 ±1000，再四舍五入交给 Motor；左右映射不会在此处混用。
  6. 第 190 行：输出成功后返回 OK。

用当前左右实测默认值把闭环算一遍

输入：左右目标都是 200mm/s，左实测 180mm/s，右实测 220mm/s，trim 都为 0；左右 Kp 都是 0.6，Ki=Kd=0。

过程：左前馈为 200×0.43114+19.884=106.112，左 P 反馈为 0.6×(200-180)=12；右前馈为 200×0.44289+22.205=110.783，右 P 反馈为 0.6×(200-220)=-12。

输出：左浮点命令约 118.112，四舍五入为 118；右浮点命令约 98.783，四舍五入为 99。同一目标速度的左右基础 PWM 本来就不同，反馈再根据各自误差增减。

边界条件：任一总和超过 ±1000 会被限幅；目标、实测或 dt 异常会停机；Odometry 标定无效也会停机。

设计原因：前馈承担大部分基础驱动力，反馈只需修正实际偏差，通常比完全依赖 PID 从零“追”目标更直接。

5. 难点函数拆解

PID_Update() 的时间意义

输入：PID 对象、目标速度、实测速度和本周期 dt。

过程：算误差 → 用 error×dt 更新积分 → 用 误差变化/dt 算微分 → P/I/D 加权 → 输出限幅。

输出：一个反馈 PWM 修正，不是最终 PWM；MotionWheel 还会加前馈与 trim。

边界条件：PID 内部只在 dt>0 时算 D；MotionWheel 更严格，dt 非有限或≤0 会直接停车。PID 本身不检查空指针，调用者必须保证有效。

设计原因：显式使用 dt，使循环偶尔累计多拍时 I/D 仍按真实经过时间计算。当前 Ki、Kd 为 0，dt 对 P 项数值没有影响，但接口为以后启用 I/D 保持正确。

为什么前馈、反馈和 trim 要分开

输入：目标轮速、编码器实测轮速、上层姿态/巡线修正。

过程：前馈解决“想达到这个速度大概要多少 PWM”；PID 解决“实际还差多少”；trim 解决“为了车身方向，两轮需要额外制造多少差异”。

输出：finalPWM = feedforward(target) + PID(target, actual, dt) + trim。

边界条件：三项相加后统一限幅。发生饱和时，外层 trim 或反馈的一部分会被截掉，因此长时间顶到 1000 通常意味着目标过高、机械阻力大或参数失配。

设计原因：把职责分开后，速度标定、左右轮差异和航向/巡线修正可以独立理解与调试。

6. 放回总体调用链

直线：Mission → MotionManager → MotionStraight → 左右相同目标速度 + 航向 trim → MotionWheel

巡线：Mission → MotionManager → MotionLine → 灰度误差换成左右不同目标速度 → MotionWheel

转向：Mission → MotionManager → Nav → 左右大小相等、符号相反的目标速度 → MotionWheel

共同闭环：MotionWheel 目标 mm/s ←→ Odometry 左右实测 mm/s → 左右 PID 修正；目标 mm/s → 前馈；三者加 trim → ±1000 PWM

硬件输出：MotionWheel → Motor_SetPWM → 左轮 B/右轮 A 接线映射 → 方向 GPIO + PWM 绝对值 → PWM duty→compare → TIMG8 C0/C1 → TB6612 → 电机

App 每个有效系统周期先调用 Odometry_Update() 得到新的左右速度，再由 MotionManager 推进当前运动模块。因此 MotionWheel 本拍读取的是刚更新的反馈，并使用同一个真实 dt 计算控制。

停止也分层处理：MotionWheel_Stop() 立即关闭 PWM 并清 PID；需要更短停车距离时，MotionManager 第 35～67 行先释放 PWM，再短时间调用 Motor_Brake()，最后必须回到 Motor_StopAll()，避免 H 桥持续短刹导致电机或驱动器发热。

7. 常见误解与调试提示

  • “Motor_Forward(200) 是 200mm/s”：错，它是开环 PWM=200。要控制 mm/s，应通过 MotionWheel/上层运动模块。
  • “两轮正 PWM 应对应相同方向引脚”：错。物理安装和接线相反，右轮在底层乘 -1；逻辑正号相同，硬件电平可以相反。
  • “PWM_SetCompareA(500) 直接写寄存器 500”：错。500 是 50% 刻度，内部换算后当前恰好写 800；函数名容易让初学者混淆。
  • “Stop 和 Brake 一样”：错。Stop 关闭 PWM/方向，倾向滑行；Brake 让 H 桥短接绕组以更快停止。
  • “PID 当前三项都在工作”：错。当前 Ki=0、Kd=0，反馈项只有 P；最终命令还会叠加左右各自的前馈与静摩擦补偿。
  • “积分上限 0 表示把积分锁死为 0”：在通用 PID_Clamp 中，limit≤0 表示不启用该限幅。不过当前 Ki=0，积分不参与输出；MotionWheel 禁止 Ki>0 时仍使用非正积分上限。
  • “Reset 已经让轮子停下”：错。Reset 只清软件状态；Stop 才同时调用 Motor_StopAll()。

调参数时的联动影响

  • 真正决定上电默认行为的是 LEFT/RIGHT 五组宏。公共 MOTION_WHEEL_KP、MOTION_WHEEL_FEEDFORWARD_PWM_PER_MMPS 等只保留兼容参考；单独改公共宏不会覆盖当前写死的左右实测默认值。
  • 修改左或右前馈/静摩擦都会影响直线、巡线和转向，因为三个行为共用 MotionWheel。应分别用两轮稳态数据标定 lwff/lwsf 与 rwff/rwsf，再调各侧 Kp。
  • 提高 Kp 会增强纠偏，也可能让左右轮抖动；启用 Ki 前必须设置正的积分上限，并观察输出饱和和停车后的积分残留。
  • 设置静摩擦补偿过大会导致低速目标一给出就突然窜动；过小则电机在低速区只响不转。
  • K1～K5 是兼容批量写入口，会让左右对应参数重新相等；已完成独立标定时应使用 K17～K26，避免误把两侧差异抹掉。
  • 修改 PWM_MAX_DUTY 或定时器 period 时，要同步检查 MotionWheel 最大命令、duty 换算和 SysConfig 的 TIMG8 配置，不能只改一个宏。
  • 左右轮方向或通道映射若改错，会让巡线修正、航向修正和 Nav 原地转向同时反常。排查时先做低 PWM 单轮测试，确认“软件左/右”和“逻辑正/负”。
  • Odometry_CountsPerMM 标定影响实测 mm/s；它偏大/偏小会让 PID 误判轮速，不能仅靠改 Kp 掩盖编码器标定错误。

8. 学完自测

1. 为什么 PWM 强度可以是无符号数，而 Motor 命令必须有正负号？

PWM 只表示输出有多强，方向由 H 桥 GPIO 单独表达；Motor 层用正负号同时表达逻辑方向，再把绝对值交给 PWM。

2. 左轮目标 200mm/s、实测 180mm/s、trim=0，使用当前左轮默认值时最终整数 PWM 约是多少？

前馈为 200×0.43114+19.884=106.112，P 反馈为 0.6×20=12，浮点总和 118.112，转换后约为 118。

3. 左右轮为什么各自需要一份 PID_t？

两轮的编码器速度、积分历史和上次误差不同；共用一份会让一侧的历史污染另一侧控制。

4. Ki 当前为 0 时，积分代码为什么还保留？启用 Ki 还要做什么？

PID 是通用实现，当前轮速配置暂时关闭 I。启用 Ki 时要给正的积分上限，实车调参并检查饱和、低速和停车切换。

5. MotionWheel 收到 NULL、非法 dt 或无效里程标定时会怎样？

未配置时直接返回 NOT_CONFIGURED；动态参数或里程异常时调用 MotionWheel_Stop，关闭电机并清 PID，再返回对应错误码。
