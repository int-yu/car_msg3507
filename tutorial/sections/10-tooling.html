2. 初学者需要先知道

  • 工具链：编译器、链接器、生成器和烧录器组成的一条加工线，把源码变成芯片可运行程序。
  • ELF 固件文件：既含机器码，也可含符号和调试信息；本工程链接输出为 ArcLineTest.out。
  • 网页串口接口：浏览器经用户授权访问本机串口。网页不能静默选择设备，必须由点击触发选择框。
  • 流：串口数据像连续水流，不保证一次 read 正好得到一整行；可能半行、多行或从任意位置切开。
  • 流锁：调用 getReader()/getWriter() 后，该对象独占 readable/writable；关闭端口前必须取消读取并释放锁。
  • 解析器：把原始文本转换为有字段名和类型的样本对象。
  • 单元测试：给一个小功能输入固定样例，自动比较实际与期望；失败时进程返回非零，适合脚本和 CI 判断。
  • Serial Wire Debug：ARM 常用双线调试/烧录接口；频率过高或廉价探针接线较差时可能不稳定。
  • 工程元数据：IDE 用 XML/JSON 保存项目名、芯片、编译参数、SDK、目标探针等。它不是业务 C 代码，但会决定如何构建。
  • 生成文件：由 SysConfig/编译器自动写出的文件，下一次生成可能覆盖手改内容；应修改源配置而不是结果。

function crc8Update(crc, data) {
    crc ^= data;
    for (let bit = 0; bit < 8; bit += 1) {
        crc = (crc & 0x80) ? (((crc << 1) ^ 0x07) & 0xFF) : ((crc << 1) & 0xFF);
    }
    return crc;
}

为什么 reader.read() 不能直接 split 当前 value

输入：Web Serial 任意长度的 Uint8Array，可能是半行，也可能包含多行和下一行开头。

过程：TextDecoder 流式解码 → 追加旧 buffer → 按换行切分 → pop 保存未闭合尾巴 → 只解析已经完整的行。

输出：不会把半截数字当成完整样本，也不会丢失跨 read 边界的内容。

边界条件：若发送端永远不发换行，buffer 会持续增长；本固件每行用 CRLF 结束。

设计原因：串口/USB 分包边界与业务行边界无关，解析器必须自己恢复完整记录。

### 4.2 `car_debug.html` 调试网页

**打开方式。** 双击 `car_debug.html` 即可，`file://` 下 Web Serial 可用（该协议属于安全上下文），不需要架本地服务器。只支持 **Chrome / Edge** 等 Chromium 内核浏览器，Firefox 与 Safari 没有实现 Web Serial API。点「连接串口」后在弹出的设备列表里选无线 DAPLink 对应的串口，波特率固定 115200 8N1。

**与固件的约定。** 网页只依赖第 3.3 节那套 ASCII 命令和第 5.7 节的遥测 CSV，不假设任何固件内部实现。因此改动固件时只要保持这两个契约不变，网页无需同步修改；反过来，一旦新增命令或调整 CSV 字段，必须同步更新第 3.3 / 5.7 节，网页按表头列名解析，不硬编码列位置。

页面服务**两个目的**：遥控采集数据（供后续视觉/决策任务使用）和闭环调参。分为八块：连接与急停、运动控制（含 `W` 恒速与 `N` 巡线）、**遥控采集**、参数面板、标定向导、调参试验、遥测与曲线、原始终端。

**工作流一：调参试验。**

1. 用「任务→对象→阶段」导航选定要调什么（如 轮速 → 左轮 → P），点「开始试验」；
2. 连接和阶段切换时，网页提前准备本阶段的**最优掩码**（调左轮只传 TL/LV/PL）并按固件 `Q` 回报的真实频率上限设频；
3. 点「开始试验」后，网页先清空本次终端与曲线、记录参数快照，然后立即发激励命令，不再等待遥测准备链；
4. 指标卡明确标注数据来源：直启流程使用 `实时流 xx Hz`，峰值可能漏采、仅供参考；若后续流程使用板载捕获，`板载 100 Hz 捕获` 表示与控制环同频、指标可信；
5. 每条记录可设为基准（虚线叠加对比）、导出 CSV，或一键复制 **AI 调参包**（含控制结构、参数、激励、列说明、自动指标和完整数据），直接粘给 AI 要下一组参数；
6. 在阶段参数区就地微调（`−`/`+` 按钮或直接输入，写入立即生效），再跑下一次对比。

**工作流二：遥控采集。** 启用键盘驾驶后用 <kbd>W</kbd><kbd>A</kbd><kbd>S</kbd><kbd>D</kbd> 驾驶（W/S 走恒速命令，A/D 走相对转角），点「开始录制」整段记录遥测流；录制中按 <kbd>1</kbd>~<kbd>5</kbd> 打场景标记（直道/弯道/路口/障碍/异常），标记写入 CSV 的 `mark` 列。停止后可导出整段 CSV，或**按标记分段导出**——每个场景一个文件，便于后续分类使用。<kbd>Esc</kbd> 全局急停。

参数面板连接后自动 `K?` 拉取全表，悬浮显示范围与上电默认值，支持存到浏览器 localStorage、下次上电一键回写（固件参数掉电不保存）。标定向导包装陀螺尺度 `E1`/`Y`/`E0` 流程和里程 `cpm` 换算写入。

几个实现上的取舍：

- **频率上限由固件说了算，网页不各存一份阈值。** 改掩码后网页主动发 `Q` 查询，用固件回报的 `MAX` 自动设频；勾选字段的瞬间另有一份本地复算用于即时显示（公式与 `Telemetry.c` 相同），两者不一致就说明固件与网页失配。
- **指标必须标注来源。** 直启流程优先让车先动，因此默认用实时流指标；实时流受串口带宽限制低于 100 Hz 控制环，阶跃峰值会落在采样点之间，界面必须让用户看见这次结论是基于哪种数据得出的。
- **捕获与实时流分开解析、分开存储。** 前缀 `CH,`/`C,` 与 `H,`/`D,` 互不冲突，两者列集和采样率都不同，混在一起会让指标取到错误的时间基准。

5. 难点函数拆解

从 CCS Build 到烧录

输入：C 源码、main.syscfg 和 CCS 工具链/SDK/链接设置。

过程：SysConfig 生成外设配置 → TI Clang 编译 → linker 生成 Debug/ArcLineTest.out → flash.ps1 检查路径 → pyOCD 经 SWD 写入 → reset。

输出：芯片运行新固件，map/linkInfo 辅助分析。

边界条件：只构建 Release、工程改名、pyOCD 不在 PATH 或探针接线失败都会中断链路。

设计原因：CCS 管复杂构建，短脚本提供稳定、可重复的外部烧录入口。

动态表头为什么可靠

输入：固件掩码变化后发出的 H 行与 D 行。

过程：H 建 columns 并清旧样本；D 校验列数，再按列名选择文本、十六进制、十进制或复合解析。

输出：任意合法字段组合都得到带名字的 sample，UI 自动重建曲线项。

边界条件：切换瞬间残行被丢弃；新增特殊列需同步类型分类与测试。

设计原因：协议以名称描述列，避免固件和网页依赖固定位置。

7. 常见误解与调试提示

  • “一次 read 就是一行”：错，串口任意分包，必须缓存半截尾巴。
  • “close 前不用管 reader”：错，reader 锁未释放时 close 会失败。
  • “Z1 和 Z2 都只是清零角度”：错。Z1 只改当前航向基准；Z2 会阻塞重采陀螺仪零漂并归零，约 0.8 秒必须保持车辆静止。
  • “曲线 x 轴按 ms”：错，当前按样本索引均匀排布，ms 保留在导出数据。
  • “Debug 生成配置可直接改”：错，应改 main.syscfg/CCS 属性后重建。
  • “README 说 25H 就一定运行 25H”：错，当前 main.c 加载 Test。
  • “测试通过代表所有边界完美”：错，测试只锁住已有场景。

修改联动

  • 新增命令：同步固件解析、网页控件/事件、README，必要时补测试。
  • 新增遥测字段：同步固件掩码/表头/数据/带宽、网页类型分类和测试样例。
  • 修改工程名或构建配置：同步 flash.ps1 的 Debug ELF 路径。
  • 修改芯片/探针：优先在 CCS 属性调整，并同步 pyOCD target。
  • 修改 main.syscfg：重新生成并 Clean/Rebuild，不改 Debug 结果文件。
  • 升级 SDK/SysConfig/TI Clang：检查 .cproject 版本、生成差异和实机行为。

8. 学完自测

1. 为什么断开时要 cancel reader 再 close？

getReader 锁住 readable；cancel 唤醒读循环，finally releaseLock 后端口才能关闭。

2. 新 H 表头为什么清空旧 samples？

新掩码改变字段，旧样本与新样本列定义不同，混合会错位。

3. 测试为什么不复制 TelemetryParser？

副本可能漂移；抽取网页真代码才能避免假通过。

4. Build 与 flash.ps1 的文件契约是什么？

Debug 配置必须生成 Debug/ArcLineTest.out ELF。

5. README、main.c、main.syscfg 冲突时怎么判断？

运行任务看 main.c，硬件源配置看 main.syscfg，README 用作索引并应修正。
