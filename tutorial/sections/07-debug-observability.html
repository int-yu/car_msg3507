模块 07：OLED、调试显示与遥测

本模块提供两扇观察窗：OLED 显示现场状态，Telemetry 把数据整理成 CSV，经 UART1 交给 car_debug.html 画曲线。

1. 先看它在整车中的位置

一句话职责
把系统内部状态转换成 OLED 像素和串口 CSV，让人能够观察、诊断和记录小车行为。
上游输入
Heading 航向、Odometry 左右路程/速度、Graydetect 五路位图、Key 按键位图、MotionManager 模式与错误、Mission 状态、K230 最近目标。
下游输出
SSD1306 的 128×64 显存数据、I2C 命令/数据帧，以及 UART1 的 H,... 表头和 D,... 数据行。
依赖模块
OLED 字模/位图、I2C DriverLib、Serial1、系统 100Hz Tick，以及各状态模块的只读查询接口。
被谁调用
App 初始化和更新流程调用 DebugDisplay 与 Telemetry；OLED 绘图函数也可被其他调试页面直接调用。
总体位置
State / Motion / Mission → DebugDisplay → OLED → I2C → 屏幕，以及 State / Motion / K230 → Telemetry → Serial1 → car_debug。

2. 初学者需要先知道

  • 显存/帧缓冲：先在 RAM 里画完整画面，再统一发送到屏幕。像先在草稿纸上排版，最后整页复印；本项目缓冲区正好 1024 字节。
  • 页寻址：SSD1306 把竖直 8 个像素装进一个字节。64 行被分成 8 页，每页 128 列。
  • I²C 总线：用 SCL 时钟线和 SDA 数据线连接 MCU 与 OLED。设备地址为 0x3C，发送前还要用控制字节区分命令与图像数据。
  • 字模：提前把字符的黑白像素编码成字节数组。显示字母不是让 OLED 理解“A”，而是从数组找到 A 的像素并复制到显存。
  • 字段掩码：用整数的每一位控制一组遥测字段是否启用；例如 0x01 开启 yaw，0x03 同时开启 yaw 与 sensor。
  • 逗号分隔值：每列之间用逗号分隔。表头告诉接收端每一列是什么，数据行必须严格使用同一顺序和列数。
  • 波特率：串口每秒传输的位数。115200 8N1 中一个数据字节在线路上通常占 10 位，所以理论上约 11520 字节/秒。

3. 文件导读

文件作用初学者阅读重点
Hardware/Display/OLED.h/.c屏幕驱动与绘图库。I2C 命令/数据、1024 字节显存、坐标转 page/bit、文本数字和几何图形。

坐标如何落到一个字节的某一位

输入：像素坐标 (x,y)，其中 x=0～127、y=0～63。

过程：page = y / 8 找到第几组竖直 8 点；bit = y & 7 得到组内第几位；字节位置是 s_buffer[page][x]。

输出：点亮使用 |= 1U<<bit，熄灭使用 &= ~(1U<<bit)，反色使用 ^= 1U<<bit。

边界条件：任何写像素操作都必须确认 x、y 在屏幕内，否则会访问显存数组之外。

设计原因：这种布局与 SSD1306 的 page 写入格式一致，整页发送时无需重新排列每个像素。

4.3 I2C 发送：等待、启动、超时和错误恢复

static uint8_t OLED_Transfer(const uint8_t *data, uint8_t length)
{
    uint32_t timeout = OLED_TIMEOUT;
    while (((DL_I2C_getControllerStatus(OLED_I2C_INST) & DL_I2C_CONTROLLER_STATUS_IDLE) == 0U) &&
           (timeout-- > 0U)) {}
    if (timeout == 0U) return 0U;

    DL_I2C_flushControllerTXFIFO(OLED_I2C_INST);
    DL_I2C_fillControllerTXFIFO(OLED_I2C_INST, data, length);
    DL_I2C_startControllerTransfer(OLED_I2C_INST, OLED_ADDRESS,
                                   DL_I2C_CONTROLLER_DIRECTION_TX, length);
    delay_cycles(16U);
    timeout = OLED_TIMEOUT;
    while (((DL_I2C_getControllerStatus(OLED_I2C_INST) & DL_I2C_CONTROLLER_STATUS_BUSY) != 0U) &&
           (timeout-- > 0U)) {}
    if ((timeout == 0U) || ((DL_I2C_getControllerStatus(OLED_I2C_INST) &
                             DL_I2C_CONTROLLER_STATUS_ERROR) != 0U))
    {
        DL_I2C_resetControllerTransfer(OLED_I2C_INST);
        return 0U;
    }
    return 1U;
}
  1. 第 18～20 行：私有发送函数接收只读数据指针和字节数，创建超时计数。
  2. 第 21～23 行：先等控制器进入 IDLE；每轮递减 timeout。耗尽就返回 0，避免死循环。
  3. 第 25 行：清空旧的发送 FIFO，避免上一次残留字节混入本包。
  4. 第 26 行：把 length 个字节填入控制器 TX FIFO。调用者必须确保 data 至少有这么长。
  5. 第 27～28 行：向地址 0x3C 发起 TX 方向传输，并告诉硬件本包长度。
  6. 第 29 行：短等 16 个 CPU 周期，让硬件状态有时间进入传输阶段。
  7. 第 30～32 行：重置超时，再等待 BUSY 清除，也就是等待本包完成。
  8. 第 33～35 行：超时或控制器 ERROR 任一发生都进入失败处理。
  9. 第 36～37 行：复位当前控制器传输状态并返回 0。
  10. 第 39～40 行：完整发送且无错误才返回 1。
static void OLED_Command(uint8_t command)
{
    uint8_t packet[2] = {0x00U, command};
    if (OLED_Transfer(packet, 2U) == 0U) s_busError = 1U;
}

static void OLED_Data(const uint8_t *data, uint8_t count)
{
    uint8_t packet[OLED_CHUNK_BYTES + 1U];
    uint8_t offset = 0U;
    packet[0] = 0x40U;
    while (offset < count)
    {
        uint8_t i;
        uint8_t chunk = (uint8_t)(count - offset);
        if (chunk > OLED_CHUNK_BYTES) chunk = OLED_CHUNK_BYTES;
        for (i = 0U; i < chunk; i++) packet[i + 1U] = data[offset + i];
        if (OLED_Transfer(packet, (uint8_t)(chunk + 1U)) == 0U)
        {
            s_busError = 1U;
            return;
        }
        offset = (uint8_t)(offset + chunk);
    }
}

static void OLED_SetCursor(uint8_t page, uint8_t x)
{
    OLED_Command((uint8_t)(0xB0U | page));
    OLED_Command((uint8_t)(0x10U | ((x & 0xF0U) >> 4)));
    OLED_Command((uint8_t)(x & 0x0FU));
}
  1. 第 42～46 行：命令包固定两个字节：控制字节 0x00 表示后面是命令，第二字节是 SSD1306 命令；发送失败便锁存总线错误。
  2. 第 48～52 行：数据发送准备 8 字节 packet，首字节 0x40 表示后续是显存数据，并从 offset=0 开始。
  3. 第 53～57 行：剩余数据可能超过 7 字节，因此每轮取“剩余量”和 7 中较小者。
  4. 第 58 行：把本块有效数据复制到 packet[1...]，保留 packet[0] 控制字节。
  5. 第 59～63 行：发送 chunk+1 字节；失败就记录错误并立即停止后续块。
  6. 第 64～66 行：offset 前进 chunk，直到 count 个数据全部发完。
  7. 第 68～70 行：设置页地址：0xB0 | page，page 应为 0～7。
  8. 第 71 行：取 x 的高 4 位，右移后并入列地址高半命令 0x10。
  9. 第 72～73 行：取 x 的低 4 位作为列地址低半命令。三个命令共同把 OLED 写指针移动到指定 page/x。

4.4 初始化、全屏刷新与局部刷新

void OLED_Init(void)
{
    static const uint8_t commands[] = {
        0xAE, 0xD5, 0x80, 0xA8, 0x3F, 0xD3, 0x00, 0x40,
        0xA1, 0xC8, 0xDA, 0x12, 0x81, 0xCF, 0xD9, 0xF1,
        0xDB, 0x30, 0xA4, 0xA6, 0x8D, 0x14, 0xAF
    };
    uint8_t i;
    Delay_ms(100U);
    s_busError = 0U;
    for (i = 0U; i < (uint8_t)sizeof(commands); i++) OLED_Command(commands[i]);
    s_ready = (s_busError == 0U) ? 1U : 0U;
    OLED_Clear();
    if (s_ready != 0U) OLED_Update();
}

uint8_t OLED_IsReady(void) { return s_ready; }

void OLED_Update(void)
{
    uint8_t page;
    if (s_ready == 0U) return;
    for (page = 0U; page < 8U; page++)
    {
        OLED_SetCursor(page, 0U);
        OLED_Data(s_buffer[page], 128U);
    }
    if (s_busError != 0U) s_ready = 0U;
}

  2. 第 82～84 行：等待屏幕上电 100ms，再清除旧总线错误。
  3. 第 85 行：sizeof(commands) 是命令数组总字节数；逐条用命令控制字节发送。
  4. 第 86 行：只要初始化期间没有任何传输失败就标记 ready。
  5. 第 87～88 行：无论 ready 与否先清 RAM 显存；只有屏幕可用时才把空白画面真正发出去。
  6. 第 91 行：单行查询函数返回 ready。
  7. 第 93～96 行：全屏更新先检查 ready，离线时直接返回，避免每次调试刷新都阻塞在坏总线上。
  8. 第 97～101 行：遍历 8 页，每页把光标设到 x=0，再发送该页 128 字节。
  9. 第 102～103 行：刷新中若任何分块传输报错，就把 ready 清零；之后的全屏 Update 将停止访问总线。
void OLED_UpdateArea(int16_t x, int16_t y, uint8_t width, uint8_t height)
{
    int16_t firstPage = y / 8;
    int16_t lastPage = (y + height - 1) / 8;
    int16_t page;
    if ((x < 0) || (x > 127) || (width == 0U) || (height == 0U)) return;
    if ((x + width) > 128) width = (uint8_t)(128 - x);
    if (firstPage < 0) firstPage = 0;
    if (lastPage > 7) lastPage = 7;
    for (page = firstPage; page <= lastPage; page++)
    {
        OLED_SetCursor((uint8_t)page, (uint8_t)x);
        OLED_Data(&s_buffer[page][x], width);
    }
}

void OLED_Clear(void) { memset(s_buffer, 0, sizeof(s_buffer)); }

void OLED_ClearArea(int16_t x, int16_t y, uint8_t width, uint8_t height)
{
    int16_t px;
    int16_t py;
    for (py = y; py < (int16_t)(y + height); py++)
        for (px = x; px < (int16_t)(x + width); px++)
            if ((px >= 0) && (px < 128) && (py >= 0) && (py < 64))
                s_buffer[py / 8][px] &= (uint8_t)~(1U << (py & 7));
}

void OLED_Reverse(void)
{
    uint16_t i;
    for (i = 0U; i < sizeof(s_buffer); i++) ((uint8_t *)s_buffer)[i] ^= 0xFFU;
}
  1. 第 105～109 行：局部更新把 y 范围换算成首尾 page。高度跨越页边界时，会发送涉及的每一整段 page 字节。
  2. 第 110 行：x 越界、宽或高为 0 时不发送。
  3. 第 111 行：右边超过 128 列时缩短 width，形成横向裁剪。
  4. 第 112～113 行：页范围裁剪到 0～7。
  5. 第 114～118 行：逐页移动光标，从 &s_buffer[page][x] 取得该页第 x 列地址，只发送 width 字节。
  6. 第 121 行：memset 把 1024 字节全设 0，只修改 RAM；需要 Update 才能让屏幕变空。
  7. 第 123～126 行：局部清除用 px/py 枚举矩形内每个像素。
  8. 第 127～130 行：只有屏内像素才修改；定位 page/bit 后，用与取反掩码把对应位清 0。
  9. 第 133～136 行：全屏反色把二维数组强制视为连续 uint8_t *，逐字节与 0xFF 异或。

void OLED_DrawPoint(int16_t x, int16_t y)
{
    if ((x >= 0) && (x < 128) && (y >= 0) && (y < 64))
        s_buffer[y / 8][x] |= (uint8_t)(1U << (y & 7));
}

uint8_t OLED_GetPoint(int16_t x, int16_t y)
{
    if ((x < 0) || (x >= 128) || (y < 0) || (y >= 64)) return 0U;
    return ((s_buffer[y / 8][x] & (1U << (y & 7))) != 0U) ? 1U : 0U;
}

void OLED_ShowImage(int16_t x, int16_t y, uint8_t width, uint8_t height, const uint8_t *image)
{
    uint8_t px;
    uint8_t py;
    if (image == NULL) return;
    OLED_ClearArea(x, y, width, height);
    for (py = 0U; py < height; py++)
        for (px = 0U; px < width; px++)
            if ((image[(py / 8U) * width + px] & (1U << (py & 7U))) != 0U)
                OLED_DrawPoint((int16_t)(x + px), (int16_t)(y + py));
}

  3. 第 155～159 行：GetPoint 越界返回 0；屏内则用按位与测试目标 bit，并规范成 1/0。

void OLED_ShowString(int16_t x, int16_t y, const char *string, uint8_t fontSize)
{
    int16_t offset = 0;

        if (((uint8_t)*string & 0x80U) != 0U)
        {
            OLED_ShowChar((int16_t)(x + offset), y, '?', fontSize);
            while ((((uint8_t)string[1] & 0xC0U) == 0x80U)) string++;
        }
        else OLED_ShowChar((int16_t)(x + offset), y, *string, fontSize);
        offset = (int16_t)(offset + fontSize);
        string++;
    }
}

{

    while (power-- > 0U) value *= 10U;

void OLED_DrawLine(int16_t x0, int16_t y0, int16_t x1, int16_t y1)
{
    int16_t dx = (int16_t)abs(x1 - x0);
    int16_t sx = (x0 < x1) ? 1 : -1;
    int16_t dy = (int16_t)-abs(y1 - y0);
    int16_t sy = (y0 < y1) ? 1 : -1;
    int16_t error = (int16_t)(dx + dy);
    for (;;)
    {
        int16_t twice;
        OLED_DrawPoint(x0, y0);
        if ((x0 == x1) && (y0 == y1)) break;
        twice = (int16_t)(2 * error);
        if (twice >= dy) { error = (int16_t)(error + dy); x0 = (int16_t)(x0 + sx); }
        if (twice <= dx) { error = (int16_t)(error + dx); y0 = (int16_t)(y0 + sy); }
    }
}

void OLED_DrawTriangle(int16_t x0, int16_t y0,
                       int16_t x1, int16_t y1,
                       int16_t x2, int16_t y2,
                       uint8_t filled)
{
    if (filled == 0U)
    {
        OLED_DrawLine(x0, y0, x1, y1);
        OLED_DrawLine(x1, y1, x2, y2);
        OLED_DrawLine(x2, y2, x0, y0);
    }
    else
    {
        int16_t x;
        int16_t y;
        int16_t minX = (int16_t)fminf((float)x0, fminf((float)x1, (float)x2));
        int16_t maxX = (int16_t)fmaxf((float)x0, fmaxf((float)x1, (float)x2));
        int16_t minY = (int16_t)fminf((float)y0, fminf((float)y1, (float)y2));
        int16_t maxY = (int16_t)fmaxf((float)y0, fmaxf((float)y1, (float)y2));
        for (y = minY; y <= maxY; y++) for (x = minX; x <= maxX; x++)
        {
            int32_t a = OLED_Edge(x0, y0, x1, y1, x, y);
            int32_t b = OLED_Edge(x1, y1, x2, y2, x, y);
            int32_t c = OLED_Edge(x2, y2, x0, y0, x, y);
            if (((a >= 0) && (b >= 0) && (c >= 0)) ||
                ((a <= 0) && (b <= 0) && (c <= 0)))
            {
                OLED_DrawPoint(x, y);
            }
        }
    }
}

void OLED_DrawCircle(int16_t x, int16_t y, uint8_t radius, uint8_t filled)
{
    int16_t px;
    int16_t py;
    int16_t rr = (int16_t)(radius * radius);
    for (py = -(int16_t)radius; py <= radius; py++) for (px = -(int16_t)radius; px <= radius; px++)
    {
        int16_t d = (int16_t)(px * px + py * py);
        if ((filled != 0U && d <= rr) || (filled == 0U && (d >= rr - radius) && (d <= rr + radius)))
            OLED_DrawPoint((int16_t)(x + px), (int16_t)(y + py));
    }
}

void OLED_DrawEllipse(int16_t x, int16_t y, uint8_t a, uint8_t b, uint8_t filled)
{
    int16_t px;
    int16_t py;
    float aa = (float)a * a;
    float bb = (float)b * b;
    for (py = -(int16_t)b; py <= b; py++) for (px = -(int16_t)a; px <= a; px++)
    {
        float d = ((float)px * px) / aa + ((float)py * py) / bb;
        if (((filled != 0U) && (d <= 1.0f)) ||
            ((filled == 0U) && (d >= 0.90f) && (d <= 1.10f)))
            OLED_DrawPoint((int16_t)(x + px), (int16_t)(y + py));
    }
}

void OLED_DrawArc(int16_t x, int16_t y, uint8_t radius,
                  int16_t startAngle, int16_t endAngle,
                  uint8_t filled)
{
    int16_t angle;
    if (endAngle < startAngle) endAngle = (int16_t)(endAngle + 360);
    for (angle = startAngle; angle <= endAngle; angle++)
    {
        float rad = (float)angle * 3.1415926f / 180.0f;
        int16_t px = (int16_t)(x + cosf(rad) * radius);
        int16_t py = (int16_t)(y + sinf(rad) * radius);
        if (filled != 0U) OLED_DrawLine(x, y, px, py);
        else OLED_DrawPoint(px, py);
    }
}

  4. 第 372～378 行：用标准椭圆方程 px²/a²+py²/b²；实心取≤1，空心取 0.90～1.10 的近似边带。

4.20 动态安全频率与模式文本

/* 根据当前字段掩码计算安全频率上限。
 * 纯整数运算，避免在 MCU 上引入浮点。
 * 公式：maxRate = (BYTES_PER_SEC * MAX_PERCENT) / (rowBytes * 100)
 * 结果夹紧到 1..TELEMETRY_RATE_HARD_LIMIT_HZ。 */
uint8_t Telemetry_GetMaxRateHz(void)
{
    uint32_t rowBytes = (uint32_t)Telemetry_EstimateRowBytes();
    uint32_t maxRate;

    maxRate = (TELEMETRY_UART_BYTES_PER_SECOND * TELEMETRY_MAX_BLOCKING_PERCENT)
              / (rowBytes * 100U);

    if (maxRate < 1U)
    {
        maxRate = 1U;
    }
    if (maxRate > (uint32_t)TELEMETRY_RATE_HARD_LIMIT_HZ)
    {
        maxRate = (uint32_t)TELEMETRY_RATE_HARD_LIMIT_HZ;
    }
    return (uint8_t)maxRate;
}

void Telemetry_Update(uint8_t elapsedTicks, uint8_t pressedKeys)
{
    uint8_t interval;

    s_elapsedMs += (uint32_t)elapsedTicks * 10U;

    if (s_rateHz == 0U)
    {
        return;
    }

    /* s_rateHz 由 SetRateHz 限制在 1..TELEMETRY_RATE_HARD_LIMIT_HZ（== TICK_HZ），
     * TICK_HZ / s_rateHz 最小为 1，interval == 0 分支不可能触发，已删除。 */
    interval = (uint8_t)(TELEMETRY_TICK_HZ / s_rateHz);

    s_tickAccumulator = (uint8_t)(s_tickAccumulator + elapsedTicks);
    if (s_tickAccumulator < interval)
    {
        return;
    }
    s_tickAccumulator = 0U;

    if (s_headerPending != 0U)
    {
        Telemetry_SendHeader();
        s_headerPending = 0U;
    }
    Telemetry_SendRow(pressedKeys);
}

5. 难点函数拆解

OLED_Update()：为什么先画 RAM 再发屏幕

输入：已经由 Clear、ShowString、DrawPoint 等函数修改好的 1024 字节 s_buffer。

过程：检查 ready → 逐 page 设置列 0 → 每页 128 字节按 7 字节分块通过 I2C 发送 → 汇总总线错误。

输出：屏幕显示一帧与 RAM 相同的画面。

边界条件：OLED 离线时不发送；任一 I2C 包失败会使后续全屏更新停止。绘图函数本身通常不自动 Update。

设计原因：多个文字和图形先在内存合成，最后统一刷新，避免用户看到“先清屏、再一行行出现”的撕裂过程。

字段掩码如何保证 H 与 D 一致

输入：s_fieldMask 的低六位。

过程：SendHeader 与 SendRow 按完全相同的位判断、完全相同的 YAW→SENSOR→DISTANCE→SPEED→MODE→K230 顺序追加内容。

输出：每个 H 列名都能与 D 中同位置的值一一对应。

边界条件：一个位可能展开两列，K230 又把三个数压进一列；接收端不能按“掩码位数量”猜列数。掩码改变后必须重发 H。

设计原因：网页以表头为协议事实，可在不修改解析器主流程的情况下选择不同字段组合。

6. 放回总体调用链

OLED 现场链：App_Update → DebugDisplay_Update(elapsedTicks) → Heading / Odometry / Graydetect / Key / MotionManager / Mission 查询 → OLED 显存绘制 → OLED_Update → I2C0 → SSD1306

遥测串口链：App_Update → Telemetry_Update(elapsedTicks, pressedKeys) → State / Motion / K230 查询 → H/D CSV → Serial1 阻塞发送 → 无线 DAPLink 串口 → car_debug.html

网页解析链：串口任意分片 → car_debug 缓冲到完整行 → H 更新 columns 并清旧样本 → D 按 columns 解析 → gray 十六进制转换 / mode 文本保留 / k230 冒号展开 → 曲线与 CSV 导出

7. 常见误解与调试提示

  • “调用 ShowString 后屏幕马上变”：错。它只改 RAM 显存；还要调用 OLED_Update 或 OLED_UpdateArea。
  • “字模里的一个字节是一横行”：错。当前格式是纵向 8 点，一个字节是一列中的 8 个像素。
  • “OLED_Data 有汉字，所以 ShowString 能显示汉字”：错。当前 UTF-8 分支只显示问号，中文查表尚未接入。
  • “OLED 上 M:IDLE 一定表示没有刹车”：错。DebugDisplay 没有单独处理 BRAKE，它会落入 IDLE；可结合 Telemetry mode 判断。
  • “请求 23Hz 就精确每秒 23 行，而且仍严格低于 20% 带宽”：错。当前整数间隔使它约为 25Hz，全字段时约占 21.3%；建议优先使用能整除 100 的频率，或修正调度算法。
  • “字段掩码一个 bit 就对应一列”：错。sensor/distance/speed 各展开两列，k230 是一个复合列。
  • “提高遥测频率只影响串口”：错。Serial1 是阻塞发送，行太长或频率太高会拖慢 App、运动控制和 OLED 更新。

修改时需要同步检查

  • 更改 OLED 分辨率、寻址模式或控制器型号时，要同步修改显存尺寸、8 页循环、坐标裁剪、初始化命令和字模布局。
  • 新增 Telemetry 字段必须同时修改字段宏、最坏行长估算、SendHeader、SendRow 和 car_debug 解析/显示；缺一处就会列错位或安全上限失真。
  • 修改 UART1 波特率应通过 SysConfig，让 BLUETOOTH_UART_BAUD_RATE 自动同步；仍需重新核对网页串口设置和实际阻塞占比。
  • 修改系统 Tick 频率时，要同步修改 TELEMETRY_TICK_HZ、elapsedMs 每拍换算、DebugDisplay 刷新拍数和所有超时常量。
  • 若实现中文显示，要决定 UTF-8/GB2312 编码、查表终止规则和未命中回退，并确保 16×16 字符横移 16 像素而非现有 fontSize 6/8。

8. 学完自测

1. 像素 (37,19) 位于显存哪一页、哪一列、哪个 bit？

page=19/8=2，列为 x=37，组内 bit 为 19&7=3，所以位置是 s_buffer[2][37] 的 bit3。

2. 为什么 OLED_Data 的 8×16 字符恰好需要 16 字节？

每字节纵向表示 8 点；字符宽 8 列、高 16 点，需要上下两页，每页 8 列，所以是 8×2=16 字节。

3. 为什么字段掩码改变后必须先发 H 表头？

掩码会改变数据列的数量和顺序；car_debug 按最近表头解释 D 行。若不重发表头，数据会进入错误列或因列数不匹配被丢弃。

4. 115200 8N1、20% 预算、全字段 98 字节时，公式为什么给 23Hz，而当前实现为什么仍可能超预算？

理论字节率为 115200/10=11520B/s，20% 是 2304B/s，2304/98 整数结果为 23。但发送间隔使用 100/23=4，实际变成约 25Hz，占用约 25×98/11520=21.3%。因此公式与整数拍调度没有形成严格闭环。

5. ShowString 遇到“你”会怎样？

当前实现识别到 UTF-8 高位字节后画一个问号，并跳过该字符的续字节；虽然数据文件含“你”的字模，但尚无查表调用。
